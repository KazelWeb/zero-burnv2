# Scripting Agent Rules

## 1. Core Constraints
- Max Lines: STRICTLY 300 lines max per script/module. No exceptions.
- Editing Mode: When editing existing code, ONLY provide Code Snippets (VSCode Copilot style) for Find & Replace. Do not output full scripts to save tokens and speed.
- Creation Mode: When creating a full system or new file, output the full script and full structure.

## 2. Coding Structure & Organization
- Naming: Name scripts strictly based on relevance.
- Architecture: Use a highly Modular, Folderized, and Organized structure in the Explorer.
- Documentation: Create ReadME files (Instructions and Overview) for documentation.
- Comments: NO COMMENTS in primary codebases. Comments are exclusively reserved for ReadME files.

## 3. Security & Exploitation Prevention
- Server Security: Server-Sided Scripts must be structured to prevent visibility or reverse-engineering by Hackers using Dex Explorer.
- Client-Server Handshake: Safeguard Client Scripts Structure. Connect Client and Server scripts securely. If a player's client behavior modifies unexpectedly and is not authorized by your LocalScripts/ServerScripts, automatically kick the exploiter.

## 4. Code Quality & Style
- Anti-Slop: NO AI-Slop code structure or naming.
- Minimalism: Keep code simple, minimal, and as few lines as possible.
- Paradigm: Use clean, standard Lua Object-Oriented Programming (OOP) with metatables (e.g., BaseEntity.__index = BaseEntity).
